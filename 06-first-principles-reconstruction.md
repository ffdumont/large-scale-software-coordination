# 6. First-Principles Reconstruction

## Research question

If existing frameworks did not exist, what minimal organization would we design for several hundred software developers?

## Method

Start from the problems identified in section 1. Introduce a practice only when a problem justifies it.

Each proposed practice must answer:

1. What problem does it solve?
2. Why does it exist?
3. What happens if it is removed?
4. What lighter alternatives exist?
5. What evidence supports it?
6. What assumptions must hold for it to work?

## Candidate minimal mechanisms

- stable product or domain teams;
- explicit ownership boundaries;
- lightweight strategy-to-execution alignment;
- periodic multi-team planning focused on dependencies and trade-offs;
- visible flow of work;
- WIP limitation;
- technical governance;
- CI/CD and integrated quality practices;
- architecture evolution loop;
- platform capabilities;
- dependency reduction mechanism;
- organizational improvement loop.

## Principle

The target is not a new framework. It is a minimal operating system for large-scale software coordination.
